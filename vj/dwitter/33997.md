## Code dé-minifié et annoté

```javascript
l = c.width = 840;  // fixe la largeur (840px) ET efface le canvas chaque frame

for (i = 20000; i--;) {   // 20 000 particules, pas de clear partiel (tout recalculé)

  // --- Paramètre de "ligne" ---
  y = i / l;              // y ∈ [0, ~23.8] — index continu sur toutes les itérations

  // --- k : oscillation modulée ---
  // Enveloppe : (4 + 3·sin(y·2 − t)) ∈ [1, 7] — undulation lente animée par t
  // Porteuse  : cos(i/29) — oscille vite (période de 182 itérations en i)
  // k ∈ [−7, 7] — définit la position latérale de chaque point
  k = (4 + S(y * 2 - t) * 3) * C(i / 29);

  // --- e : position "verticale" dans un référentiel interne ---
  // y ∈ [0, 23.8] → e ∈ [−11, −8] (toujours négatif)
  e = y / 8 - 11;

  // --- d : distance radiale pondérée (exposant 0.66 ≈ ⅔) ---
  // (k² + e²)^0.66 ∈ [~16, ~30] — joue le rôle de "rayon" dans l'espace paramétrique
  d = (k * k + e * e) ** 0.66;

  // --- q : amplitude de déplacement (terme clé de la forme organique) ---
  // • S(k*2)          : oscillation fine sur k
  // • 0.3/k           : singularité en k=0 → "épine" lumineuse au centre de chaque bras
  // • S(y/25)*k*(...)  : modulation basse fréquence × k, amplitude 9+4·sin(...)
  q = S(k * 2) + 0.3 / k + S(y / 25) * k * (9 + 4 * S(e * 9 - d * 3 + t * 2));

  // --- P : angle de rotation différentiel ---
  // Chaque point a son propre angle qui évolue avec d → torsion organique
  P = d - t;

  // --- Dessin : 1×1 px, couleur héritée (pas de fillStyle explicite) ---
  x.fillRect(
    q * S(P) + d * 39 - l / 2,   // X : position orbitale + offset radial − 420 (centrage)
    q + 30 * C(P) + l / 4,       // Y : position + oscillation verticale + 210 (centrage)
    1, 1
  );
}
```

## Analyse visuelle

### Principe général

**20 000 particules 1×1px** forment une créature marine organique. Le canvas est effacé à chaque frame (`c.width=840`) → rendu propre, pas de traîne.

### Structure de la forme

| Variable | Rôle | Plage |
|----------|------|-------|
| `y` | Index de parcours (pseudo-ligne) | [0, 23.8] |
| `k` | Position latérale oscillante | [−7, 7] |
| `e` | Coordonnée interne (toujours neg.) | [−11, −8] |
| `d` | Rayon paramétrique (stable) | [~16, ~30] |
| `q` | Amplitude de déplacement | [−100?, +100?] |
| `P` | Angle de rotation propre à chaque point | `d−t` |

### Anatomie de la créature

1. **Corps principal** : la variation lente de `k` avec `sin(y·2−t)` crée des épaisseurs ondulantes
2. **Tentacules/bras** : `cos(i/29)` fait osciller rapidement k → plusieurs bras superposés
3. **Épine dorsale** : `0.3/k` → singularité quand `k ≈ 0` → ligne de lumière intense
4. **Mouvement** : `P = d − t` donne à chaque point une rotation individuelle → torsion fluide
5. **Respiration** : `sin(y·2−t)` dans l'enveloppe de k → ondulation globale au rythme de t

### Coordonnées écran

- **X** = `q·sin(d−t) + d·39 − 420`
  - `d·39 ∈ [624, 1170]` → moins 420 → [204, 750] : centre horizontal variable selon d
  - `q·sin(P)` : oscillation orbitale autour de ce centre
- **Y** = `q + 30·cos(d−t) + 210`
  - Centré à 210 (= l/4), orbite verticale de rayon 30 px + déplacement q

### Résolution et canvas

- Largeur forcée : **840 px** (non-standard dwitter)
- Hauteur : non touchée (default navigateur, ~540 ou 1080)
- Pas de `fillStyle` explicite → couleur par défaut (noir sur fond transparent/blanc page)

## Traduction Python prévue

### Particularités à adapter

- **Canvas 840px** → normaliser à la largeur du rendu cible (`w/2` pour centrage)
- **Clear chaque frame** : `c.width=840` → `img = Image.new("RGBA", (w,h), (0,0,0,0))`
- **20 000 itérations séquentielles** (k dépend de i via C(i/29) — non vectorisable directement)
  - Optim partielle : calculer d'abord tous les `y`, `k`, `e`, `d` en numpy vectorisé
  - `q` et `P` dépendent de k et d → numpy aussi
  - Seul le `fillRect 1×1` est un scatter-assign numpy
- **Singularité `0.3/k`** : protéger contre k=0 avec `np.where(np.abs(k) > 1e-6, 0.3/k, 0)`
- **`d*39 − l/2`** → adapter le facteur 39 à la résolution : `d * 39 * w / 840`

### Vectorisation numpy possible

```python
i = np.arange(N-1, -1, -1, dtype=np.float32)   # i de N-1 à 0
y = i / N * (N / l)                              # équivalent i/840
k = (4 + np.sin(y*2 - t)*3) * np.cos(i/29)
e = y/8 - 11
d = (k*k + e*e) ** 0.66
k_safe = np.where(np.abs(k) > 1e-4, k, 1e-4)
q = np.sin(k*2) + 0.3/k_safe + np.sin(y/25)*k*(9 + 4*np.sin(e*9 - d*3 + t*2))
P = d - t
px = (q*np.sin(P) + d*39 - 420) * w/840 + w/2    # à calibrer
py = (q + 30*np.cos(P) + 210)                      # à calibrer
```

### Adaptation couleur

- Remplacer le noir par défaut par les couleurs de la palette active
- Possibilité de colorer par valeur de `d` (plus loin = couleur différente) ou par `k`
- L'épine dorsale (k≈0, q grand) peut avoir une teinte distincte
