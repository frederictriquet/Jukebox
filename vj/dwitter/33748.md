## Code dé-minifié et annoté

```javascript
function u(t) {
  // Efface le canvas uniquement à t=0 (!t = 1 seulement quand t === 0)
  // → après la première frame, pas de clear → les traces s'accumulent
  x.fillRect(0, 0, 2000, 2000 * !t);

  // 5000 particules, i descend de 4999 à 0
  // La commande fillRect est dans le post-corps du for (après la virgule)
  for (i = 5000; i--; x.fillRect(
    960 + 9 * s * C(i),   // x = centre + rayon * cos(i)   [i en radians]
    540 + 9 * s * S(i),   // y = centre + rayon * sin(i)
    s,                     // largeur du carré
    s                      // hauteur du carré
  )) {
    // c ∈ [-99, 99], varie avec le temps ET avec i²
    // i*i crée une distribution non linéaire des phases → aspect "cristal"
    c = 99 * S(t + i * i);

    // Couleur dominante bleu-vert (B ≈ 4c, G ≈ 3c, R ≈ c)
    // Alpha très faible (0.07) → accumulation de traces longuettes
    x.fillStyle = R(c, c * 3, c * 4, 0.07);

    // s = taille de la particule ET rayon orbital
    // Deux rôles : contrôle la taille du carré ET la distance au centre
    // (i/9 + t*7) % 138 → s oscille entre 0 et 137 en spirale temporelle
    s = (i / 9 + t * 7) % 138;
  }
}
```

## Analyse visuelle

### Principe géométrique

Chaque particule `i` est placée en **coordonnées polaires** :
- **Angle** = `i` radians (non normalisé, tourne sur lui-même)
- **Rayon** = `9 * s` pixels, où `s = (i/9 + t*7) % 138`

La taille de la particule est `s×s` pixels, identique au rayon divisé par 9 → les particules lointaines sont grandes, les proches sont petites.

### Animation

- `t*7` dans `s` fait tourner/pulser la spirale à vitesse fixe
- `S(t + i*i)` dans la couleur : le terme `i*i` crée une interférence non-linéaire qui donne un aspect irisé, "cristallin"
- Pas de clear après t=0 : les anciennes frames restent avec alpha cumulé → **traînes longues**

### Palette de couleurs

`R(c, c*3, c*4, 0.07)` avec `c ∈ [-99, 99]` :
- Quand `c > 0` : tons bleu-vert (B > G > R)
- Quand `c < 0` : canvas clamp à 0 → noir (pixels sombres)
- Alternance rythmée par `sin(t + i²)` → effet pulsé chromatique

### Résolution originale

Canvas 1920×1080, centre à (960, 540). Les particules peuvent aller jusqu'à `9 * 137 = 1233px` du centre → débordent largement.

## Traduction Python prévue

### Adaptation pour VJingLayer

- Coordonnées polaires centrées : `cx = w/2`, `cy = h/2`
- Échelle : `radius_scale = 9 * min(w,h) / 1080` (normalisé par rapport à l'original)
- Pas de clear entre frames : utiliser alpha faible sur img existante (ou fond semi-transparent)
- `N = 5000` iterations (séquentiel, comme l'attractor)
- Taille particule : `s = (i/9 + t*7) % max_s` avec `max_s` proportionnel à résolution
- Couleur : depuis la palette active plutôt que le bleu-vert codé en dur

### Particularité à gérer

La double boucle for (corps + post-incrément) se traduit par un calcul de `s` **avant** le `fillRect` de même itération, ce qui en Python donne un simple `for` avec calcul de `s` puis dessin.
